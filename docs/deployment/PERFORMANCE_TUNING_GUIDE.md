app/core/cache_optimization.py\nclass CacheOptimizer:\n    def calculate_ttl(self, hit_rate: float) -> int:\n        \"\"\"Calculate TTL based on hit rate.\"\"\"\n        base_ttl = 3600  # 1 hour\n        \n        if hit_rate > 0.95:  # High hit rate\n            return int(base_ttl * 1.5)  # Increase by 50%\n        elif hit_rate < 0.70:  # Low hit rate\n            return int(base_ttl * 0.5)  # Decrease by 50%\n        else:\n            return base_ttl\n```\n\n### Cache Warming\n\nPre-load frequently accessed data:\n\n```python\nclass CacheWarmer:\n    async def warm_cache(self):\n        \"\"\"Pre-warm cache with frequently accessed data.\"\"\"\n        # Warm tier configurations\n        tiers = await self.db.query(SubscriptionTier).all()\n        for tier in tiers:\n            await self.cache.set(f\"tier:{tier.id}\", tier, ttl=3600)\n        \n        # Warm feature configurations\n        features = await self.db.query(Feature).all()\n        for feature in features:\n            await self.cache.set(f\"feature:{feature.id}\", feature, ttl=3600)\n```\n\n### Cache Invalidation\n\nSmart invalidation on data changes:\n\n```python\nclass CacheInvalidationPolicy:\n    async def on_tier_change(self, user_id: str):\n        \"\"\"Invalidate cache on tier change.\"\"\"\n        await self.cache.delete(f\"user_tier:{user_id}\")\n        await self.cache.delete(f\"user_features:{user_id}\")\n    \n    async def on_feature_change(self, feature_id: str):\n        \"\"\"Invalidate cache on feature change.\"\"\"\n        await self.cache.delete(f\"feature:{feature_id}\")\n        # Invalidate all user feature caches\n        await self.cache.delete_pattern(\"user_features:*\")\n```\n\n### Monitoring Cache in users:\n    tier = user.tier  # Separate query for each user\n\n# Good: Eager loading\nusers = db.query(User).options(\n    joinedload(User.tier)\n).all()\n```\n\n**Batch Queries**:\n```python\n# Bad: Multiple queries\nfor user_id in user_ids:\n    user = db.query(User).filter(User.id == user_id).first()\n\n# Good: Batch query\nusers = db.query(User).filter(User.id.in_(user_ids)).all()\n```\n\n### Response Optimization\n\n**Compression**:\n```python\n# Enable gzip compression\nfrom fastapi.middleware.gzip import GZIPMiddleware\n\napp.add_middleware(GZIPMiddleware, minimum_size=1000)\n```\n\n**Field Selection**:\n```python\n# Allow clients to select fields\n@router.get(\"/users/{user_id}\")\nasync def get_user(user_id: str, fields: str = None):\n    user = db.query(User).filter(User.id == user_id).first()\n    \n    if fields:\n        selected_fields = fields.split(\",\")\n        return {field: getattr(user, field) for field in selected_fields}\n    \n    return user\n```\n\n**Pagination**:\n```python\n# Limit response size\n@router.get(\"/users\")\nasync def list_users(skip: int = 0, limit: int = 100):\n    if limit > 1000:\n        limit = 1000  # Max 1000 items\n    \n    return db.query(User).offset(skip).limit(limit).all()\n```\n\n### Response Caching\n\n```python\nclass ResponseCache:\n    async def get_cached_response(self, key: str):\n        \"\"\"Get cached response.\"\"\"\n        return await self.cache.get(key)\n    \n    async def cache_response(self, key: str, response: dict, ttl: int = 300):\n        \"\"\"Cache response.\"\"\"\n        await self.cache.set(key, response, ttl=ttl)\n\n# Usage\n@router.get(\"/api/tiers\")\n@cache_response(ttl=3600)\nasync def list_tiers():\n    return db.query(SubscriptionTier).all()\n```\n\n---\n\n## Database Optimization\n\n### Index 200ms\n\n**Solution**:\n```bash\n# 1. Check cache hit rate\ncurl http://localhost:8000/api/metrics/cache\n\n# 2. If hit rate < 85%, warm cache\npython scripts/deployment/warm_cache.py\n\n# 3. Check database performance\nEXPLAIN ANALYZE SELECT * FROM subscription_tiers;\n\n# 4. If slow, add index\nCREATE INDEX idx_tier_id ON subscription_tiers(id);\n```\n\n### Low Cache Hit Rate\n\n**Problem**: Cache hit rate < 70%\n\n**Solution**:\n```bash\n# 1. Check cache size\nredis-cli INFO memory\n\n# 2. If full, increase cache size\n# Update Redis configuration\n\n# 3. Check TTL settings\n# Increase TTL for frequently accessed data\n\n# 4. Warm cache\npython scripts/deployment/warm_cache.py\n```\n\n### High API Latency\n\n**Problem**: API response time > 500ms\n\n**Solution**:\n```bash\n# 1. Check database queries\nEXPLAIN ANALYZE SELECT * FROM users WHERE id = 'xxx';\n\n# 2. If slow, add index\nCREATE INDEX idx_user_id ON users(id);\n\n# 3. Check API response size\ncurl -i http://localhost:8000/api/users | grep Content-Length\n\n# 4. If large, enable compression\n# Already enabled in app\n\n# 5. Check for N+1 queries\n# Use eager loading\n```\n\n### High Memory Usage\n\n**Problem**: Memory usage > 80%\n\n**Solution**:\n```bash\n# 1. Check cache size\nredis-cli INFO memory\n\n# 2.
