## 📝 Documentation Standards

### File Naming
- Use `SCREAMING_SNAKE_CASE.md` for documentation files
- Use descriptive names that indicate content
- Prefix with category when helpful (e.g., `API_GUIDE.md`)

### Structure
- Start with title and metadata (date, status, version)
- Include table of contents for long documents
- Use clear headings and sections
- Add code examples where relevant
- Include links to related documentation

### Maintenance
- Update `Last Updated` date when making changes
- Archive completed work to `archives/history/`
- Keep active docs in root or category folders
- Remove outdated information promptly
