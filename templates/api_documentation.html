Introduction

The Vrenum API allows you to programmatically create SMS and voice verifications for 1,800+ services across 180+ countries. Our RESTful API uses JSON for requests and responses, making it easy to integrate with any programming language.

Base URL: https://vrenum.app/api
API Version: v1
Format: JSON

Authentication

All API requests require authentication using an API key. You can generate API keys from your dashboard. Include your API key in the Authorization header as a Bearer token.

Example Request

bash 
curl -X GET "https://vrenum.app/api/verify/history" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
python 
import requests

headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

response = requests.get(
    "https://vrenum.app/api/verify/history",
    headers=headers
)

print(response.json())
javascript 
const response = await fetch('https://vrenum.app/api/verify/history', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data);

Rate Limits

API rate limits vary by tier to ensure fair usage and system stability.

Tier Requests/Minute Requests/Hour Concurrent Verifications
Freemium 10 100 2
Pay-As-You-Go 30 500 5
Pro 60 2000 10
Custom 120 5000 25
Rate Limit Headers: Check X-RateLimit-Remaining and X-RateLimit-Reset headers in responses.

Create Verification

POST /api/verify/create

Create a new SMS or voice verification for a specific service and country.

Request Body

Parameter Type Required Description
service_id string Yes Service identifier (e.g., "telegram", "whatsapp")
country_code string Yes ISO 3166-1 alpha-2 country code (e.g., "US", "GB")
verification_type string No Type: "sms" or "voice" (default: "sms")
area_code string No Specific area code (Pro+ only)

Example Request

{
  "service_id": "telegram",
  "country_code": "US",
  "verification_type": "sms"
}

Example Response

{
  "success": true,
  "verification_id": "ver_abc123xyz",
  "phone_number": "+1234567890",
  "service": "telegram",
  "country": "US",
  "status": "pending",
  "expires_at": "2024-01-15T10:30:00Z",
  "cost": 0.85
}

Check Status

GET /api/verify/status/{verification_id}

Check the current status of a verification.

Response

{
  "verification_id": "ver_abc123xyz",
  "status": "completed",
  "phone_number": "+1234567890",
  "messages_received": 1,
  "created_at": "2024-01-15T10:00:00Z",
  "completed_at": "2024-01-15T10:02:30Z"
}

Get Messages

GET /api/verify/{verification_id}/messages

Retrieve all messages received for a verification.

Response

{
  "verification_id": "ver_abc123xyz",
  "messages": [
    {
      "id": "msg_xyz789",
      "text": "Your verification code is: 123456",
      "code": "123456",
      "received_at": "2024-01-15T10:02:30Z"
    }
  ]
}

Cancel Verification

POST /api/verify/{verification_id}/cancel

Cancel a pending verification and receive a refund.

Response

{
  "success": true,
  "verification_id": "ver_abc123xyz",
  "status": "cancelled",
  "refund_amount": 0.85,
  "refund_status": "completed"
}

Webhooks

Webhooks allow you to receive real-time notifications when verifications are completed or messages are received.

Configure webhooks in your Webhook Settings

Webhook Payload

{
  "event": "verification.completed",
  "verification_id": "ver_abc123xyz",
  "phone_number": "+1234567890",
  "status": "completed",
  "messages": [
    {
      "text": "Your code is: 123456",
      "code": "123456"
    }
  ],
  "timestamp": "2024-01-15T10:02:30Z"
}

Error Codes

Code Status Description

402 Payment Required Insufficient credits
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error, contact support
