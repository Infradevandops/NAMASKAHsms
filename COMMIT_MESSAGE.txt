fix: permanent solution for i18n translation key regression on dashboard

Implemented comprehensive i18n-aware DOM update system to prevent
translation keys from appearing after dynamic content loads.

Problem:
- Dashboard showed correct translations initially
- After ~200ms, text reverted to raw keys (dashboard.title, tiers.current_plan)
- Caused by race condition between i18n and dynamic content loaders

Solution (Option B - Permanent Fix):
- Added MutationObserver to auto-translate dynamically added content
- Created helper utilities for safe DOM manipulation
- Fixed all dynamic loaders to preserve i18n system
- Exposed window.i18n and window.i18nReady globally
