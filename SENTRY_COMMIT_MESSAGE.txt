feat: integrate Sentry error tracking and monitoring

## Summary
Integrated Sentry for production error tracking, performance monitoring,
and real-time alerting. Addresses the need for proactive error detection
beyond CI/CD testing.

## Changes

### Core Integration
- Added Sentry configuration fields to `app/core/config.py`
  - sentry_dsn
  - sentry_traces_sample_rate (default: 0.1)
  - sentry_profiles_sample_rate (default: 0.1)

- Fixed `app/core/sentry.py` to use get_settings()
  - Updated to use proper config attribute names
  - Maintained all existing integrations (FastAPI, SQLAlchemy, Redis)

- Initialized Sentry in `app/core/lifespan.py`
  - Runs on application startup
  - Placed before database init for early error capture

- Updated `.env.example` with Sentry DSN

### Documentation
- Created `docs/SENTRY_SETUP.md` - Comprehensive setup guide
- Created `docs/SENTRY_DEPLOYMENT_CHECKLIST.md` - Step-by-step deployment
- Created `docs/SENTRY_QUICK_REFERENCE.md` - Quick operations reference
- Created `docs/SENTRY_INTEGRATION_SUMMARY.md` - Integration overview
- Updated `README.md` with Sentry status and links
